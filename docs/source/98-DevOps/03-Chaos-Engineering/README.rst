DevOps Best Practice in Chaos Engineering
==============================================================================

混沌工程是 Netflix 公司在 all in AWS 并全面使用 microservice 模式之后, 总结出的一套系统工程方法论. 用于解决在有成千上万个微服务的情况下, 如何使用混沌测试实验, 在 **生产环境中** 进行测试, 随机造成破坏. 使得整体服务稳定, 高可用.
